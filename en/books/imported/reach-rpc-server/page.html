The Reach RPC Server provides access to compiled JavaScript backends via an HTTPS-accessible JSON-based RPC protocol. The server allows frontends to be written in any programming language. Reach provides client libraries for JavaScript, Python, and Go. It is easy to implement a client library yourself. An example frontend written using the Reach RPC Server is shown in the tutorial section on RPC-based frontends.

  30. All `/backend` methods are interactive RPC methods that accept three arguments:
  31. + `ctcId` --- A contract RPC handle to provide as the contract to the backend
  32. + `values` --- An object containing the non-function components of the participant interact interface of the backend.
  33. + `methods` --- An object whose keys correspond to the function components of the participant interact interface of the backend, but whose values are `true`.
  36. As a special case, if `values` contains `stdlib.hasRandom` bound to `true`, then the [JavaScript standard library](##ref-frontends-js)'s implementation of `hasRandom` is provided to the backend.
  38. As the backend executes, any of the components of `methods` invoked will be executed as interactive RPC callbacks as described by the [Reach RPC Protocol Specification](##ref-backends-rpc-proto).
  39. Reach RPC Client libraries _should_ expose a function that hides the details of the construction of the `values` and `methods` objects and implements interactive RPC callback handlers automatically.
  41. + `/kont` handles interactive RPC continuation completion during an interactive RPC method.
  42. It should not be invoked directly by frontends.
  44. + `/stop` quits the server.
  47. ## Reach RPC Client Standard Options
  49. Reach RPC client libraries must accept a dictionary data structure with the following keys for customizing their behavior:
  51. + `host` --- This value sets the hostname to contact for the Reach RPC Server instance.
  52. If it is not present, the client library must default to the value of the environment variable `REACH_RPC_SERVER`.
  53. + `port` --- This value sets the TCP port to contact for the Reach RPC Server instance.
  54. If it is not present, the client library must default to the value of the environment variable `REACH_RPC_PORT`.
  55. + `verify` --- This value determines whether to verify the TLS certificate of the Reach RPC Server instance.
  56. If it is not present, the client library must default to the value of the environment variable `REACH_RPC_TLS_REJECT_UNVERIFIED`.
  57. If that is not present, it must default to true.
  59. To disable verification, set this value to the string: `"0"`; any other value will be considered to mean "enable".
  60. + `timeout` --- This value sets the number of seconds to wait for the Reach RPC Server instance to respond to its first request.
  61. If it is not present, the client library must default to the value of the environment variable `REACH_RPC_TIMEOUT`.
  62. If that is not present, it must default to 5 seconds.
  63. + `key` --- This value sets the API key for the Reach RPC Server instance.
  64. If it is not present, the client library must default to the value of the environment variable `REACH_RPC_KEY`.
